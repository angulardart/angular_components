date-input

This directive globs onto a material-input and automatically tries to parse dates on change events (enter or blur). It recognizes a bunch of localized formats:

  • 7/20/2015 (or 20/7/2015, depending on locale)
  • Mon, 7/20/2015 (or Mon, 20/7/2015)
  • Jul 20, 2015
  • Mon, Jul 20, 2015
  • July 20, 2015
  • Monday, July 20, 2015
  • 2015-07-20 (ISO format)

When a valid date is entered, this directive will reformat the input. It defaultly chooses the "Jul 20, 2015" format, but this can be changed using the [date-format] property.

If the input date has a two-digit year, then the input will try to guess what the input actually means. E.g. 8/3/89 would get interpreted as August 3, 1989. (Until some point in the distant future, when it'll start interpreting that as 2089).
